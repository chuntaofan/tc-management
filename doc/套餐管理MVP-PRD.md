## 3. 需求背景

### 背景

TeleAgent 商业化套餐需要一个运营后台能力，用于维护对外售卖的套餐商品。当前项目已具备套餐列表、新增、编辑、上下架、上架时间等基础能力，但引入“待上架”概念后，必须明确待上架套餐生效后替换哪个旧套餐，否则会出现同一套餐位下多个版本并列生效的问题。

本 MVP 聚焦套餐管理的最小可用闭环：运营可以创建待上架套餐，指定其替换对象；系统在到达上架时间后自动完成版本替换；已生效或历史套餐不可直接编辑，只能通过复制生成新版本后重新发布。

### 目标用户

主要用户为 TeleAgent 运营管理员，核心场景包括：

- 配置个人套餐、企业团购、加油包的基础售卖信息。
- 为已存在套餐创建新版本。
- 管理套餐上架、下架、待上架状态。
- 避免误改已上线套餐影响线上用户或历史订单。

### 现状问题

当前套餐管理如果只依赖“上架时间”和“状态”，无法表达“待上架套餐取代哪个旧套餐”。同时，已上架套餐如果允许直接编辑，会造成线上商品规则变化不可追溯；已下架套餐如果允许重新上架，也会绕过新版本发布流程，导致历史套餐再次进入售卖状态。

### 产品方案

通过引入“套餐售卖位/系列”和“替换对象”机制，将套餐管理从单条记录上下架升级为简单版本发布模型。待上架套餐必须绑定同类型的替换对象；到达上架时间后，新套餐自动变为已上架，被替换套餐进入已被替换状态。已上架、已下架、已被替换套餐均不可编辑，运营只能复制旧套餐生成新的待上架版本。

## 4. 预估收益

### 定性收益

- 降低运营误操作风险，避免直接修改已上线套餐。
- 明确套餐版本关系，解决“待上架套餐上线后替换谁”的业务歧义。
- 保留历史套餐状态，便于后续订单、权益、审计追溯。
- 为后续接入真实后端、订单系统、权益履约系统预留清晰模型。

### 定量收益

当前 MVP 暂无真实线上数据。后续可通过套餐配置错误率、版本发布耗时、上线后回滚次数等指标评估。

## 6. 需求详情

### 6.1 业务流程

1. 运营进入 `计费管理 > 套餐管理`。
2. 点击新增套餐，或在历史套餐上点击复制。
3. 如果设置未来上架时间，套餐进入待上架状态。
4. 待上架套餐必须选择一个同类型的替换对象。
5. 到达上架时间后，新套餐状态变为已上架，被替换套餐状态变为已被替换。
6. 已上架、已下架、已被替换套餐不能编辑。
7. 已下架套餐不能重新上架；如需再次发布，运营点击复制，生成新的待上架版本。

### 6.2 套餐状态规则

| 状态 | 含义 | 编辑 | 下架/取消上架 | 重新上架 | 复制 |
| --- | --- | --- | --- | --- | --- |
| 待上架 pending | 已配置未来上架时间，尚未生效 | 支持 | 支持取消上架 | 不适用 | 不支持 |
| 已上架 active | 当前可售版本 | 不支持 | 支持下架 | 不适用 | 不支持 |
| 已下架 inactive | 已人工下架 | 不支持 | 不支持 | 不支持 | 支持 |
| 已被替换 replaced | 被新版本替换的历史版本 | 不支持 | 不支持 | 不支持 | 支持 |

### 6.3 替换对象规则

- 仅待上架套餐需要选择替换对象。
- 替换对象必须与当前套餐类型一致。
  - 个人套餐只能替换个人套餐。
  - 企业团购只能替换企业团购。
  - 加油包只能替换加油包。
- 替换对象仅展示当前已上架套餐。
- 选择替换对象后，新套餐继承其售卖位 `seriesKey`。
- 到达上架时间后，同一 `seriesKey` 下旧 active 变为 replaced，新 pending 变为 active。

### 6.4 新增套餐

入口：套餐管理页右上角 `新增套餐`。

字段规则：

| 字段 | 规则 |
| --- | --- |
| 套餐名称 | 必填 |
| 套餐类型 | 必填，支持个人套餐、企业团购、加油包 |
| 替换对象 | 当上架时间为未来时间时必填，并按套餐类型过滤 |
| 关联 SKU | 可选，后续接业务支撑系统时建议必填 |
| 上架时间 | 必填，未来时间生成待上架套餐 |
| 有效期 | 必填，支持自然月、自然年、指定天数 |
| 刊例价/实际售价 | 个人套餐、加油包必填 |
| 起价/起订数量 | 企业团购必填 |
| 权益说明 | 可配置多条 |

提交结果：

- 上架时间为未来：生成待上架套餐。
- 上架时间为当前或过去：生成已上架套餐，提交后不可编辑。
- 若待上架套餐未选择替换对象，提交失败。
- 若替换对象类型不一致，提交失败。

### 6.5 编辑套餐

入口：列表中仅待上架套餐展示 `编辑`。

编辑规则：

- 仅待上架套餐可编辑。
- 已上架、已下架、已被替换套餐不展示编辑入口。
- API 层也需校验，绕过前端编辑非待上架套餐时返回错误。
- 编辑待上架套餐时，可调整名称、价格、有效期、权益说明、上架时间、替换对象。

### 6.6 下架与取消上架

入口：列表操作列。

规则：

- 已上架套餐展示 `下架`。
- 待上架套餐展示 `取消上架`。
- 已下架套餐不允许重新上架。
- 已被替换套餐不允许上下架。

提交结果：

- 已上架套餐点击下架后，状态变为已下架。
- 待上架套餐点击取消上架后，状态变为已下架。
- 已下架套餐如需再次发布，只能走复制流程。

### 6.7 复制套餐

入口：已下架、已被替换套餐展示 `复制`。

规则：

- 点击复制后打开新增套餐弹窗。
- 默认带入原套餐名称、类型、SKU、价格、有效期、权益说明等配置。
- 名称默认追加“副本”。
- 上架时间默认设置为当前时间 + 1 天。
- 复制提交后生成新套餐记录，不修改原套餐。

### 6.8 默认数据规则

当前 MVP 默认数据：

- 个人套餐：保留体验版、基础版、标准版、旗舰版。
- 企业团购：仅保留 1 条企业团购套餐。
- 加油包：仅保留 1 条加油包套餐。
- 加油包有效期固定为 30 天。
- 默认数据版本升级后，mock localStorage 重置为最新默认数据。
